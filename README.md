一、项目核心定位
专为视障人士打造的AI辅助App，核心分为出行模式和会视模式两大独立模块，满足「出行导航」和「日常场景识别」两大核心需求，UI风格统一、交互逻辑适配视障用户（语音优先、高对比度、大操作区）。

二、两大核心模式功能定义
1. 出行模式（替代原参考图「导航」Tab）
核心目标：为视障用户提供「导航+实时出行指引」的全链路出行辅助
功能模块	详细说明
第三方导航接入	支持接入百度地图、高德地图等，获取路线规划、实时定位、导航指令（如“前方50米左转”），可独立使用纯AI指引（无导航时仅做障碍物预警）
实时AI出行指引	调用大模型API，基于摄像头画面+定位数据生成语音指引：<br>• 障碍物预警（如“前方是笔记本电脑，请停止前进”）<br>• 导航融合播报（将导航指令与场景结合，如“前方路口左转，注意左侧行人”）<br>• 路况分析（识别人行道、电梯、楼梯等场景并播报）
快速分析（双模式通用）	一键触发，简洁、准确、快速播报当前场景核心信息（出行模式示例：“前方3米有台阶，右侧有自行车，沿人行道直行”）
语音交互	底部「按住说话」支持语音指令（如“导航到公司”“调整预警距离”），同时支持键盘打字输入

2. 会视模式（替代原参考图「识别」Tab）
核心目标：为视障用户提供「眼前场景/物品的语音讲解」，满足日常识别、学习、探索需求
功能模块	详细说明
实时AI场景/物品讲解	调用大模型API，基于摄像头画面生成详细语音讲解，覆盖全场景：<br>• 物品类：工具、家电、文具（识别+用途+操作提示）<br>• 自然类：植物、动物、矿物（识别+科普）<br>• 人文类：文物、画作、书籍（解读+背景知识）<br>• 日常场景：超市、厨房等环境识别
快速分析（双模式通用）	一键触发，简洁播报核心信息（区别于详细讲解，示例：“眼前是一盆绿萝绿植”）
阅读文字功能	专门针对书籍、菜单、路牌等文字场景，OCR识别+语音朗读，适配视障阅读需求
语音交互	底部「按住说话」支持语音指令（如“讲解这个物品”“读这段文字”），同时支持键盘打字输入
持续识别	支持持续播报摄像头画面变化，适合视障用户探索环境
